💡 各平台配置获取指南：

🥜 坚果云 官方文档 ↗ 应用密码创建 ↗
  • WebDAV 地址：https://dav.jianguoyun.com/dav/邮箱/
  • 创建应用密码：登录坚果云 → 账户信息 ↗ → 安全选项 → 第三方应用管理 → 添加应用密码
  • 用户名：坚果云注册邮箱；密码：生成的应用专用密码（不是登录密码）
🏠 群晖 NAS 官方文档 ↗ WebDAV Server 帮助 ↗
  • 启用 WebDAV：DSM → 套件中心 → 搜索「WebDAV Server」→ 安装 → 启用 HTTP/HTTPS
  • HTTPS 端口 5006，HTTP 端口 5005（建议 HTTPS）
  • 地址格式：https://NAS域名:5006/webdav/（需确保 NAS 外网可达）
  • 用户名/密码：群晖系统账号（需有对应共享文件夹的读写权限）
☁️ NextCloud 官方文档 ↗
  • WebDAV 地址：https://域名/remote.php/dav/files/用户名/
  • 在 NextCloud 文件页面 → 设置 → WebDAV 可查看完整地址
  • 如启用了 2FA，需在个人安全设置中创建应用密码
  • 子目录安装：https://域名/nextcloud/remote.php/dav/files/用户名/
⚙️ GitHub Actions 配置
  • 保存到云端后，将 CONFIG_URL 和 CONFIG_AUTH 添加到仓库 Settings → Secrets
  • 签到脚本优先从云端拉取配置，失败则 fallback 到 NEWAPI_ACCOUNTS

⚠️ CORS 跨域限制解决方案：

  • 方案一（推荐）：本地打开 HTML 文件（file:// 协议），CORS 限制自动绕过
  • 方案二：群晖/NextCloud 在服务端添加 CORS 头 Access-Control-Allow-Origin
  • 方案三：坚果云不支持自定义 CORS，请用方案一
  • 方案三（最快）：使用内置 CORS 代理，下列代理任选其一，无需修改服务端配置
    • 👉 AllOrigins（api.allorigins.win）— 免费，选这个先试
    • 👉 CORSProxy（corsproxy.io）— 免费，HTTPS
    • 👉 CodeTabs（api.codetabs.com）— 免费，首次可能需等几秒
    • 👉 ThingProxy（thingproxy.freeboard.io）— 免费
    • ⚠️ CORS Anywhere（herokuapp.com）— 需先打开其首页点 Request temporary access 激活
  • ⚠️ CORS 代理意味着账号密码会经第三方转发，请勿用于敏感数据账号

📋 导入 JSON 配置

💡 使用提示：

  • 站点分组：同一站点的 URL 和 CF Clearance 只需填写一次，站点下可添加多个账号
  • 账号信息：每个账号只需填写 Session、用户ID 和备注名称
  • 保存到本地：将配置保存到浏览器本地存储，下次访问自动加载
  • 明文存储：数据以明文形式存储在浏览器中，请注意安全
